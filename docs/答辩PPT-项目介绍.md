## 一、项目与作业背景

| 项目名称 | 易宿酒店预订平台（移动端） |
|----------|----------------------------|
| 作业名称 | 第五期前端训练营大作业 |
| 开发周期 | 1/29 - 2/26（共四周） |
| 提交截止 | 2/26 20:00 |
| 答辩时间 | 3/2 - 3/5，汇报 10 分钟 + 评委提问 15-20 分钟 |

**项目定位**：易宿酒店预订平台由**移动端用户预订流程**与 **PC 端管理系统**两部分组成；本仓库为**用户端移动端**，实现酒店查询、列表筛选与详情预订核心流程，与 Node.js 后端及 PC 管理端联调对接。

---

## 二、技术栈与选型（符合考核要求）

| 考核要求 | 本项实现 |
|----------|----------|
| 前端：React / Taro / 原生小程序 / **APP** | **React Native**（APP），支持 Android / iOS |
| 服务端 | **Node.js** 后端，RESTful API，联调时使用 docker-compose 启动（自动迁移与数据填充） |
| UI 库 | 允许使用开源 UI 组件库；项目中使用 **Phosphor 图标**、**react-native-svg**、**react-native-calendars** 等，其余为自定义组件 |

**技术栈明细**：

- **框架**：React Native 0.76 + React 18.3
- **语言**：TypeScript
- **导航**：React Navigation 6.x（Stack + Bottom Tabs）
- **请求**：Axios 封装 + 统一 baseURL、拦截器
- **状态**：Zustand（搜索条件、筛选状态等）
- **能力**：定位（@react-native-community/geolocation）、日历（react-native-calendars）、列表分页与上滑加载

---

## 三、功能模块与考核要求对照

大作业要求移动端包含**酒店查询页（首页）、酒店列表页、酒店详情页**，本项实现与要求一一对应如下。

### 3.1 酒店查询页（首页）

| 考核要求 | 实现情况 |
|----------|----------|
| Banner 跳转 | 首页 Banner 轮播，支持点击跳转酒店详情 |
| 核心查询区域 | 定位、搜索框、日历组件、筛选条件、快捷标签 |
| 定位 | 定位获取当前城市并展示，支持切换城市 |
| 搜索 | 关键字搜索，跳转列表页并带参 |
| 日历 | 入住/离店日期选择，与列表页联动 |
| 筛选 | 星级、价格等筛选，条件写入全局 store，列表页消费 |
| 快捷标签 | 国内/海外、民宿/钟点房等标签，带条件进入列表 |

### 3.2 酒店列表页

| 考核要求 | 实现情况 |
|----------|----------|
| 条件筛选头 | 城市、日期、关键字等筛选头，可点击弹层修改 |
| 上滑自动加载 | 分页请求，上滑加载更多 |
| 列表项信息展示 | 酒店图、名称、星级、价格、设施标签等 |

### 3.3 酒店详情页

| 考核要求 | 实现情况 |
|----------|----------|
| 图片 Banner 滚动 | 大图左右滑动轮播 |
| 基础信息呈现 | 名称、设施、地址等 |
| 日历与间夜选择 | 入住/离店选择，与后端约定日期格式 |
| 房型价格列表 | 房型列表，价格从低到高排序展示 |

## 六、后端联调与交付物

- **后端**：Node.js 服务，使用 **docker-compose** 启动；启动时**自动执行数据库迁移与数据填充**，无需单独跑 migration/seed。
- **前端配置**：`src/constants/index.ts` 中 `API_BASE_URL` 指向后端地址（如 `http://10.0.2.2:3000/api`），与后端服务一致即可联调。
- **提交内容**：由队长提交题目名称、介绍、成员分工、**代码地址**、**线上演示地址**、**汇报 PPT** 等。

### 线上演示地址（移动端 APP 怎么填）

本仓库为 **React Native 原生 APP**，没有像网页那样的「可访问 URL」，可按以下方式之一作为「线上演示地址」提交（建议先与训练营/评委确认是否接受）：

| 方式 | 说明 | 提交内容示例 |
|------|------|----------------|
| **演示视频链接** | 录屏展示首页→列表→详情完整流程，上传后提交视频链接 |  B 站 / 腾讯视频 / 阿里云盘 等可公开访问的链接 |
| **APK 下载链接** | 打 Android 安装包，上传到网盘或 GitHub Releases，评委可安装体验 | `https://github.com/xxx/repo/releases` 或 网盘分享链接 + 说明「Android 手机/模拟器安装此 APK 体验」 |
| **答辩时现场演示** | 若主办方允许「现场/会议内共享屏幕演示」即可不填真实 URL | 在提交处填写「答辩时共享屏幕演示」，或附说明 |

- **APK 打包命令**（在项目根目录）：`cd android && .\gradlew.bat assembleRelease`（Release）或 `assembleDebug`（Debug），产出在 `android/app/build/outputs/apk/`。
- **推荐**：演示视频链接准备简单、不依赖评委设备，多数作业接受；若要求「可安装体验」再提供 APK 下载链接。
